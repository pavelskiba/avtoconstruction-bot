# Avtoconstruction Bot

Воронка для @Avtoconstruction_bot:

1. Триггер по словам "смета"/"тендер" (без учёта регистра) → приветствие.
2. Запрос на подписку на канал с кнопкой «Я подписался».
3. Проверка подписки через Telegram API (`getChatMember`). Не подписан —
   повторяет запрос. Подписан — присылает одно сообщение: ссылку на
   бесплатное вводное видео, заглушку гайда и кнопку «Ознакомиться
   подробнее», которая ведёт на лендинг.

Больше бот ничего не делает — оплата и выдача курса происходят вне бота,
на лендинге.

## Установка

```
pip install -r requirements.txt
```

## Настройка

- `.env` → `BOT_TOKEN` — токен бота (уже вписан).
- Остальное (канал, ссылка на видео, лендинг, текст-заглушка гайда) — в `config.py`.

`.env` в `.gitignore` — не публикуйте и не коммитьте этот файл.

## Запуск

```
python bot.py
```

## Деплой на Railway

1. Установи Railway CLI и залогинься:
   ```
   npm i -g @railway/cli
   railway login
   ```
2. Подключи сервис к GitHub-репозиторию проекта (Source → GitHub Repo).
3. **Volume для базы** (иначе `bot.db` будет стираться при каждом деплое):
   на канвасе проекта → ПКМ по сервису → **Attach Volume** → Mount Path `/data`.
4. Задай переменные окружения (Variables):
   ```
   BOT_TOKEN=токен_из_BotFather
   DB_PATH=/data/bot.db
   ```
5. Деплой произойдёт автоматически при пуше в GitHub (или `railway up`).

Публичный домен (Networking → Generate Domain) этому боту не нужен — он не
принимает никаких вебхуков, только long-polling к Telegram.

## Логика состояний пользователя

Хранится в SQLite (`bot.db`), поле `state`:

`new` → `waiting_subscribe` → `guide_sent`
